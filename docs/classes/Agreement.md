# Information technology - City data model - Part 1: Foundation level concepts - Agreement Pattern

This ontology specifies the foundation-level concepts related to the agreement pattern of the city data model.

This pattern imports the following files:

- [https://w3id.org/citydata/part1/v1/AgentPattern/](https://w3id.org/citydata/part1/v1/AgentPattern/)

This pattern consists of the following classes:

- [Agent](Agent.md)
- [Agreement](Agreement.md)
- [Agreement Thing](AgreementThing.md)
- [Atomic Agreement](AtomicAgreement.md)
- [Complex Agreement](ComplexAgreement.md)
- [Conjunctive Agreement](ConjunctiveAgreement.md)
- [Disjunctive Agreement](DisjunctiveAgreement.md)
This module defines the following properties:

- [agentInvolvedIn](../properties/agentInvolvedIn.md)
- [AgreementObjectProperty](../properties/AgreementObjectProperty.md)
- [commitsToActivity](../properties/commitsToActivity.md)
- [establishedOn](../properties/establishedOn.md)
- [hasClaim](../properties/hasClaim.md)
- [hasDuty](../properties/hasDuty.md)
- [hasNoClaim](../properties/hasNoClaim.md)
- [hasPrivilege](../properties/hasPrivilege.md)
- [hasSubAgreement](../properties/hasSubAgreement.md)
- [involvesAgent](../properties/involvesAgent.md)
- [validFor](../properties/validFor.md)
- [validIn](../properties/validIn.md)


The formal definition of this pattern is available in TURTLE Syntax in two files, the [core semantics](../agreement-pattern.ttl) and the SHACL [restrictions](../agreement-shacl.ttl).
